Contribute
We welcome PRs. Keep changes small, follow the release plan, and explain your impact clearly.
How to submit a PR
  • Fork the repo and create a feature branch
  • Open an Issue first for big changes
  • Use clear commits and include screenshots for UI
  • Follow existing code style and naming
  • Update changelog if behavior changed
Good first contributions
  • Fix small UI bugs
  • Add translations (English/Khmer)
  • Improve docs and examples
  • Add tests for edge cases
Suggested PR format
Title: Fix: [short summary] What changed: - ... - ... Why: - ... Testing: - macOS: - Windows (if relevant): - Screenshots:
Replace YOUR_ORG with your real GitHub org in links.
